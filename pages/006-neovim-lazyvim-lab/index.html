The NeoVim Mental Model

Unlike many modern editors that use a simple "one file = one tab" model, NeoVim separates the concepts of what you are editing from how you are viewing it.

Buffers

An in-memory representation of a file. When you open a file, it loads into a buffer. You can have many buffers open without seeing them.

:ls List buffers
:b <N> Go to buffer

Windows

A viewport onto a buffer. You can split a window to view multiple buffers at once, or the same buffer in multiple windows.

<C-w> s Split horiz
<C-w> v Split vert

Tabs

A collection of windows. Think of them as workspaces or layouts, rather than file tabs in Chrome.

:tabnew New tab
gt Next tab

LazyVim as a Distribution Layer

LazyVim is a Neovim setup powered by lazy.nvim to make it easy to customize and extend your configuration. It provides a solid default configuration for modern development.

  • Plugin Manager: Uses lazy.nvim to lazy-load plugins only when needed, making startup blazing fast.
  • Leader Key Workflows: LazyVim maps many powerful actions behind the <Space> key (the default leader). Pressing Space opens a helpful menu (which key).
  • LSP & formatting pre-configured: Out of the box, you get auto-completion, diagnostics, and code formatting hooks.

Common Friction Points & Solutions

We've all been there. Here is how to solve the most frequent issues.

The Problem: You open a file, but your fuzzy finder or LSP isn't picking up your project root properly.

The Solution: LazyVim has a concept of a "root directory". Check your current root with :pwd. You can also press <Space> f R to explore files from the project root. If you need to manually change it, use :cd %:p:h to change directory to the current file's folder.

The Problem: You navigate to a deeply nested file, but Neo-tree (the file explorer) is still showing the top-level directory and hasn't expanded to your current file.

The Solution: Press <Space> e to open/focus Neo-tree. If it hasn't focused the current file, use <Space> f E to open Neo-tree specifically at the current file.

The Problem: You forgot the keybinding to format a file, run a test, or toggle the terminal.

The Solution: The which-key plugin is your best friend. Just press <Space> and wait half a second. A menu will appear at the bottom of the screen showing all available commands. You can also search for keymaps using Telescope: <Space> s k.

The Problem: You open a Python/TS/Rust file and get no autocomplete, diagnostics, or formatting.

The Solution: First, run :LspInfo to check if the language server is attached to the current buffer. If not, you might be missing the server. Open Mason with :Mason (or <Space> c m) to easily install LSPs, formatters, and linters.

The Problem: You opened 10 files, closed one window, and now you have no idea where your files went.

The Solution: Remember that closing a window (:q) does not close the buffer. To see all open buffers, press <Space> , (switch buffers) or <Space> b b. To actually close a buffer and remove it from memory, use <Space> b d (buffer delete).

The Problem: After adding a custom plugin snippet, LazyVim throws errors on startup.

The Solution: Run :Lazy to open the lazy.nvim dashboard. Check for errors, updates, or perform a Shift+S to sync/install plugins. Review your lua/plugins/*.lua files to ensure you're returning a valid table format.
