Neo-Obsidian Drill Console

Obsidian knowledge + Neovim repetition = practical editor fluency.

Welcome to the training cockpit. You understand Obsidian deeply. You're starting out with LazyVim, but the sheer volume of keymaps and plugins feels overwhelming.

This page bridges the gap. We will translate your existing mental models of markdown vaults into concrete, daily Neovim reps.

We treat Neovim as a lighter-weight alternative to the Obsidian Electron app. Focus on daily reps instead of abstract Vim drills.

The Stack Ladder

To master Neovim for knowledge work, learn the layers in this specific order. Do not try to learn everything at once. Notably, the obsidian.nvim plugin is an enhancement layer at the top, not the foundational layer.

1

Neovim fundamentals

Basic movements, modes (Insert, Normal, Visual), opening/saving/quitting.

2

Markdown files

Editing text, navigating headings, and understanding how Neovim sees plain text files.

3

Obsidian vault structure

Navigating directories, grepping across folders, finding specific notes.

4

LazyVim conveniences

Using the core Which-Key maps (<leader>f, <leader>s) to speed up vault operations.

5

obsidian.nvim (Later Layer)

Following links, templates, backlinks. Do this after steps 1-4 are muscle memory.

Daily Drill Board

Do these concrete exercises daily until they are muscle memory.

Drill 1: Open vault from terminal
Goal: Enter your vault without using the Obsidian app.
  1. cd ~/path/to/vault
  2. nvim .
  3. Find and open a note from the file explorer.
Why: This replaces clicking the Obsidian desktop icon.
Drill 2: Find a note
Goal: Jump to a specific file instantly.
  1. Press <leader> f (Find file)
  2. Type part of the note's name
  3. Press Enter to open it
  4. Press <leader> b to return to previous buffer
Why: Replaces Obsidian's Quick Switcher (Ctrl+O).
Drill 3: Grep the vault
Goal: Search for text inside all your notes.
  1. Press <leader> / (Grep root dir)
  2. Type a tag (e.g. #project) or a phrase
  3. Select a match and hit Enter
Why: Replaces Obsidian's global search (Ctrl+Shift+F).
Drill 4: Edit frontmatter
Goal: Update metadata quickly.
  1. Press gg to jump to the top of the file
  2. Move inside the YAML frontmatter block
  3. Change a status or tag, e.g. status: draft
  4. Type :w and Enter to save
Why: Markdown metadata is powerful, but only if it's easy to edit.
Drill 5: Wiki-link surgery
Goal: Edit existing links.
  1. Find a [[wiki link]]
  2. Move cursor inside the brackets
  3. Use ci[ to change everything inside the brackets, or cw to change one word
Why: Normal mode text objects make editing links faster than using a mouse.
Drill 6: Split-based reading
Goal: View two notes side-by-side.
  1. Press <leader> w v to split vertically
  2. Use <leader> f to open a different note in the new split
  3. Press <C-h> and <C-l> to move between them
  4. Press <leader> w c or :q to close the split
Why: Essential for synthesizing ideas from multiple sources.
Drill 7: Quickfix search review
Goal: Review multiple search results sequentially.
  1. Grep for something using <leader> /
  2. Instead of hitting Enter on one result, press <C-q> to send all results to Quickfix
  3. Use <leader> x q to open the list
  4. Use ]q and [q to jump between matches
Why: Crucial for refactoring tags or reviewing all notes on a topic.
Drill 8: Daily note rep
Goal: A full workflow loop.
  1. Create or open today's daily note (e.g. 2023-10-27.md)
  2. Add one log line: - 10:00 AM: Did the Neovim drills
  3. Add one task: - [ ] Review notes later
  4. Type :wq to save and quit
Why: Proves you can use Neovim for actual daily work.

Friction-to-Rep Translator

When you feel frustrated, you are usually missing a specific mental model translation. Use this lookup table to find the drill that fixes the friction.

"I don't know what key to use"
→
Open which-key (<leader>) and inspect only Tier 1
"I lost the file I was editing"
→
Do Drill 2 (Buffers) — <leader> b b
"I need to find a thought across the vault"
→
Do Drill 3 (Grep) — <leader> /
"Obsidian is too heavy right now"
→
Do Drill 1 (Terminal open)
"obsidian.nvim is broken/confusing"
→
Ignore it. Stick to plain markdown workflows.
"LazyVim feels too magical"
→
Decode one leader group at a time.

The obsidian.nvim Later Layer

This plugin is excellent, but it is not required for immediate Neovim fluency. It is valuable later for following links, backlinks, daily note creation, and templates.

Only map these once standard markdown editing is effortless.

  • <leader> oo — open current note in Obsidian app
  • <leader> on — new note
  • <leader> od — daily note
  • <leader> os — search notes
  • <leader> ol — follow link
  • <leader> ob — backlinks
  • <leader> ot — tags

"Strip Later" Checklist

LazyVim is a starting point. As you grow comfortable, disable plugins you don't use. Here is a guide on what to keep, ignore, or evaluate later.

Keep

  • File navigation
  • Grep
  • Buffers
  • Windows
  • Git
  • Diagnostics

Ignore (Disable)

  • Debuggers
  • AI assistants
  • Extra UI toggles
  • Plugin/admin surfaces

Evaluate Later

  • Harpoon
  • obsidian.nvim
  • Advanced search plugins
  • Periodic note integrations
