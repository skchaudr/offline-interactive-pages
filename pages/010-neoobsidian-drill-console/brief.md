# Neo-Obsidian Drill Console

**Concept**: An interactive learning document that helps users learn Neovim by anchoring the practice inside Obsidian-style markdown/vault workflows. It maps existing Obsidian mental models into practical Neovim/LazyVim drills.

**Goal**: Turn existing Obsidian knowledge into daily Neovim fluency by building confidence through repeated daily reps inside a markdown vault.

**Audience**: A beginner to LazyVim but an advanced/fluent Obsidian user. They want to use Neovim as a lighter-weight alternative to the Obsidian app and need to translate their markdown vault workflows into Neovim keystrokes. They are currently struggling with the size of the LazyVim which-key map.

**Data Shape**: A series of distinct sections containing conceptual ladders, interactive component (which-key decoder), repeatable drills with persistent local state, and reference checklists.

**Required Sections**:
1. Hero / Mission ("Obsidian knowledge + Neovim repetition = practical editor fluency.")
2. The Stack Ladder (1. Neovim fundamentals, 2. Markdown files, 3. Obsidian vault structure, 4. LazyVim conveniences, 5. obsidian.nvim)
3. Which-Key Decoder (Interactive filter for "Core now", "Ignore for now", "Possible clutter")
4. Daily Drill Board (8 distinct drills with goals, steps, keystrokes, and completion tracking)
5. Friction-to-Rep Translator (Mapping frustrations to specific drills)
6. Obsidian.nvim Later Layer (Explaining why the plugin is for later, listing future workflows)
7. Seven-Day Starter Plan (Day-by-day learning sequence)
8. “Strip Later” Checklist (Keep, Ignore, Evaluate later)
