Git Rebase

A rebase essentially takes a set of commits, "copies" them, and drops them down somewhere else. While it sounds scary, it's just a way to rewrite your commit history to make it cleaner and more linear.

Why Rebase?

Instead of merging changes and creating a "merge commit" that clutters history, rebasing rewrites the project history by creating brand new commits for each commit in the original branch.

Linear History
Cleaner Commits
Easier Debugging

Interactive Rebase

Interactive rebase (git rebase -i) is the swiss army knife of Git. It lets you alter commits as they are being moved to the new base.

Common Commands

  • pick Use the commit as is.
  • reword Use the commit, but edit the commit message.
  • squash Use the commit, but meld it into previous commit.
  • drop Remove the commit entirely.

Interactive Rebase Example

Imagine you have three messy commits you want to clean up before merging.

pick 1a2b3c4 Add login feature
reword 5d6e7f8 Fix typo in login
drop 9a8b7c6 WIP testing stuff

When you save and close the editor, Git will process these commands from top to bottom.

Conflict Flow

Conflicts happen during a rebase when the changes you made in your branch overlap with the changes made in the new base.

When a conflict occurs:

Git will pause the rebase and ask you to fix the conflict manually. The process stops at the specific commit causing the issue.

CONFLICT (content): Merge conflict in index.html
error: could not apply f2... Update header

Resolution Steps

1. Open the conflicting files and resolve the markers.

Remove the <<<<<<<, =======, and >>>>>>> markers, leaving only the desired code.

2. Stage the resolved files.
3. Continue the rebase.
Pro Tip: If you get overwhelmed, you can always abort the rebase and return to your original state.

Reflog Recovery

Messed up a rebase? Don't panic. Git never truly deletes commits immediately. You can find them using the reflog.

1. Find the old state

Run git reflog to see a history of where HEAD has been.

a1b2c3d HEAD@{0}: rebase finished: returning to refs/heads/feature
d4e5f6g HEAD@{1}: rebase: checkout main
9a8b7c6 HEAD@{2}: commit: WIP testing stuff (Before rebase!)

2. Reset back to safety

Once you find the hash from before the rebase started (e.g., 9a8b7c6 or HEAD@{2}), hard reset to it.
