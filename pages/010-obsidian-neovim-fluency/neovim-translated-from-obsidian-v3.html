Neovim,
Translated from Obsidian

Part 1 for the Obsidian power user who already understands vaults, linked thinking, quick switch, backlinks, plugins, and daily notes — and now needs Neovim to stop feeling like a wall of strange keybindings.

01

What You Already Know That Transfers

The first move is not memorization. It is mapping familiar intentions to unfamiliar mechanisms.

Vault

Obsidian taught you that a folder can become a thinking environment. Neovim keeps the folder, but exposes lower-level primitives: files, buffers, windows, searches, and commands.

Obsidian instinct

You already trust search more than folder browsing. Quick switch, backlinks, and command palette made navigation feel like recall.

Neovim translation

Neovim turns those recall surfaces into pickers: searchable lists that can open files, jump to grep matches, run commands, show keymaps, or preview backlinks.

The unit is still text. A note is still a markdown file. Links are still characters on disk. This matters because the work is not being moved into a foreign database or locked app surface.

Obsidian gives polished app commands. Neovim gives composable editor commands. The difference shows up when a one-off note edit becomes a repeated transformation across many notes.

The prize is not being a keyboard wizard. The prize is repeatability: change the current word, repeat it; search the vault, preview before opening; select a paragraph, transform it.

The rosetta for the rest of this artifact is simple: Obsidian gave you the mental model for a markdown knowledge environment; Neovim gives you the grammar for moving through and rewriting that environment. You are not starting from zero. You are translating a mature workflow into a lower-level, more composable interface.

02

Neovim Vocabulary You Need First

This section is deliberately layered. Open only what you need, then come back when a word starts appearing everywhere.

Modes are states of intention.

Normalcompose actions

Delete, change, yank, jump, repeat.

Insertwrite text

The closest mode to ordinary typing.

Visualselect text

Make a range visible, then act on it.

Commandinvoke editor

Run commands, searches, substitutions.

Normal mode is where editing becomes compositional: daw means delete a word; ci( means change inside parentheses. The point is not memorizing letters; the point is forming editable sentences.
Composition slider
one commandoperator

Start with an operator: d means delete, but by itself it is waiting for a target.

Buffer ≠ window

A buffer is an open file-like object.

A window is a viewport onto a buffer. Closing a window is like closing a pane, not necessarily destroying the file or buffer. This distinction is why splits become powerful instead of chaotic.

Split ≠ tab

A split is a layout decision.

Tabs in Vim-like editors are closer to layout pages than browser tabs. Splits are the normal way to see multiple buffers at once: daily note left, source note right, search below.

Leader ≈ command namespace

Leader keys organize custom actions.

<leader> is commonly Space in modern configs. Think of it as a command palette prefix that can become muscle memory.

Lua ≈ editable ecosystem

Modern Neovim plugins are usually Lua-first.

Vimscript powered the old ecosystem. Lua made Neovim configuration and plugins feel more like ordinary programming, which is why tools such as Telescope, Snacks, and modern UI plugins feel cohesive.
03

The Translation Console

Choose an Obsidian habit. The page rewrites it into a Neovim mechanism, then shows the felt difference.

The point of this console is not to flatten Neovim into Obsidian. It is to preserve the intention while changing the machinery. Quick switch is still “open the thing I mean.” Vault search is still “find where I said this.” Backlinks are still “show incoming edges.” The Neovim version feels different because the surface is more primitive and more chainable.

Use this as a diagnosticWhen Neovim feels alien, ask: what Obsidian action am I trying to perform? Then identify whether the Neovim shape is a picker, buffer move, split action, mode change, or plugin command.

The Picker Pattern — Protagonist of Part 1

A picker is not just a search box. It is a reusable interface for turning vague intent into a selected object.

Telescope-style picker simulation

Preview

Type in the picker. Select a result. Notice the sequence: candidates appear, ranking changes, preview gives context, then one object becomes the target.

Finder

what list?

Sorter

what rises?

Previewer

what context?

Actions

what happens?

Finder: the part that says “what are we searching over?” Files, buffers, grep results, commands, backlinks, templates, and keymaps all become candidate lists.

Why pickers become everywhere

Quick switch

:Telescope find_files or a LazyVim-style file picker replaces browsing.

Vault search

Live grep turns the whole vault into a searchable candidate stream, not just a results page.

Command palette

Commands, keymaps, recent files, buffers, and diagnostics all share the same interaction pattern.

Telescope became one of the defining Lua-era Neovim plugins because it made this pattern explicit: picker, finder, sorter, previewer. Snacks.picker, fzf-lua, mini.pick, and native vim.ui.select all orbit the same idea from different angles. Once this clicks, Neovim stops feeling like a pile of unrelated plugins and starts feeling like a set of reusable surfaces.

05

obsidian.nvim — The Bridge Plugin

The bridge plugin matters because it lets your vault remain your vault while Neovim becomes the editing surface.

<leader>oo

Open an Obsidian command/menu surface. The grouping says: all Obsidian actions live under the same mnemonic namespace.

<leader>oq

Quick switch notes. The habit maps directly to Obsidian’s quick switch, but the implementation usually routes through your picker.

<leader>os

Search the vault. Same intent as Obsidian search; different feel because the result can be previewed and opened as part of editor flow.

<leader>od

Open today’s daily note. The daily note remains the starting node, but now it can sit beside source notes in splits.

<leader>on

Create a new note. This keeps note creation in the editor instead of breaking flow to reach for another app surface.

<leader>ob

Backlinks. Incoming edges become a navigable list rather than a side panel you visually scan.

<leader>ot

Templates. Template insertion becomes one part of a larger editing grammar: insert, jump, modify, repeat.

<leader>ol

Link under cursor or link selection, depending on mode/config. The important concept is link action follows cursor context.

Source postureTreat these as README/example/common-pattern bindings, not universal law. The stable concept is the <leader>o namespace: Obsidian actions grouped together so which-key and muscle memory can both find them.
