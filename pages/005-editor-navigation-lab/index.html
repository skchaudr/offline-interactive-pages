Modal Editor Navigation

Mastering movement, selection, and editing workflows.

Modal Philosophy

Traditional text editors are always in Insert Mode. Every keystroke types a character. Modal editors (like Vim or Helix) separate editing from navigation.

Normal Mode

The default mode. Keys act as commands. w moves forward a word, d deletes.

Focus: Movement as intent.

Insert Mode

Entered via commands like i (insert) or a (append). Keys type characters.

Focus: Inputting text.

Press Esc to return to Normal mode.

Basic Navigation (Vim Motions)

Keep your hands on the home row. Movement commands are designed to be composable.

  • h Left
  • j Down
  • k Up
  • l Right
  • w Next word (start)
  • e Next word (end)
  • b Previous word (start)
  • W, E, B Same, but delimited by whitespace only.
  • 0 Start of line
  • ^ First non-blank character of line
  • $ End of line
  • gg First line of file
  • G Last line of file

Action → Object vs Selection-First

A key difference in modern modal editors is the order of operations.

Vim Model (Verb + Noun)

Action -> Text Object

To delete a word: d then i w

Feedback: Invisible until the action completes.

Helix/Kakoune Model (Noun + Verb)

Select Text Object -> Action

To delete a word: m i w then d

Feedback: Text highlights before you delete it.

Text Objects

Text objects allow you to operate on semantic chunks of text rather than character by character. They are structured as i (inner) or a (around), followed by a boundary.

  • iw inner word
  • aw around word (includes following space)
  • i( or i) inner parentheses
  • i" inner quotes
  • ip inner paragraph

Example: Changing text inside quotes. Use ci" (change inner quote). This deletes the content and drops you into Insert Mode.

Advanced Workflows

Marks and Jumps

Navigate large files efficiently.

  • ma Set mark 'a'
  • 'a Jump to mark 'a'
  • Ctrl+o Jump back in jumplist
  • Ctrl+i Jump forward in jumplist

Search Navigation

  • / Search forward
  • ? Search backward
  • n Next match
  • N Previous match
  • * Search word under cursor

The Dot Command

. repeats the last edit. This is incredibly powerful when combined with search. Search for a word, change it with cw, then press n. to find the next occurrence and repeat the change.
